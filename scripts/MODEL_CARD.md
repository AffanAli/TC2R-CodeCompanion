# Model Card

## Model Name

Sentence-level document relevance classifier for extracted document context.

## Version

1.0.0

## Owner

Affan Ali

## Intended Use

Identify the most relevant sentence from each PDF or HTML document and extract the number of days if present.

## Inputs

- PDF or HTML documents
- OCR text derived from PDFs
- Sentence embeddings generated by `paraphrase-MiniLM-L6-v2`

## Outputs

Per document:

- filename
- sentence_index
- sentence_text
- probability
- parsed_days

## Model Details

- Embedding model: `sentence-transformers/paraphrase-MiniLM-L6-v2`
- Classifier: pickled gradient boosting classifier stored as `ml_classifier_gbc.pkl`
- Thresholding: predictions below threshold are masked

## Training Summary

The classifier expects sentence embeddings as numeric vectors and predicts relevance probability for each sentence. The highest-probability sentence per document is retained.

## Limitations

- OCR errors may reduce downstream accuracy.
- Day parsing currently extracts the first explicit day reference only.
- Written numbers beyond ninety are not currently handled.
- PDFs require Tesseract installed in the environment.

## Evaluation Guidance

Check:

- selected sentence relevance
- parsed day count correctness
- false positives on irrelevant documents
- stability across OCR quality bands

## Risks

- OCR noise can create incorrect day values
- threshold tuning affects recall/precision tradeoff
- embedding model choice may underperform on domain-specific text

## Operational Notes

- Output can be saved to Excel, pickle, or SQLite.
- Use unit tests and coverage before deployment.
